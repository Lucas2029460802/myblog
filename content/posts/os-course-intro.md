## 全书知识地图

操作系统课程通常围绕八大模块展开：

1. **操作系统概述** — 概念、特征、发展历程、运行环境
2. **进程管理** — 进程与线程、调度、同步与死锁
3. **内存管理** — 连续/分页/分段、虚拟内存与页面置换
4. **文件管理** — 逻辑结构、目录结构、物理分配
5. **设备管理** — I/O 控制方式、缓冲与假脱机
6. **磁盘调度** — FCFS、SSTF、SCAN、C-SCAN
7. **文件系统** — 组织、存储、检索与保护
8. **系统观** — 四大模块如何协同工作

## 操作系统是什么？

操作系统（OS）是**管理计算机硬件与软件资源的系统软件**，向上为应用程序提供接口，向下控制 CPU、内存、外设等硬件，是用户与机器之间的桥梁。

### 四大特征

| 特征 | 含义 |
|------|------|
| 并发 | 宏观上多任务同时进行 |
| 共享 | 资源被多个进程共同使用 |
| 虚拟 | 物理资源抽象为逻辑资源 |
| 异步 | 进程以不可预知的速度推进 |

### 用户态 vs 内核态

OS 通过**特权级隔离**保护内核。用户程序运行在用户态，需要访问硬件或敏感资源时，通过**系统调用、中断、异常**切换到内核态，执行完毕后再返回。

## 进程管理

- **进程**：资源分配的基本单位，有独立地址空间
- **线程**：CPU 调度的基本单位，共享进程资源，切换开销更小
- **经典调度算法**：FCFS、SJF、优先级、时间片轮转
- **死锁**：四个必要条件（互斥、占有且等待、不可抢占、循环等待），银行家算法用于避免死锁
- **PV 操作**：信号量机制是进程同步的核心，需重点掌握生产者-消费者、读者-写者、哲学家就餐三类经典问题

## 内存管理

内存管理方式不断演进：

> 连续分配 → 分页 → 分段 → 段页式

**虚拟内存**借助请求分页/分段，让程序逻辑地址空间大于物理内存。常见页面置换算法：

- OPT（最优，理论参考）
- FIFO
- LRU（最近最久未使用）
- CLOCK（近似 LRU，实用性强）

## 设备与磁盘

设备管理负责 I/O 的分配与控制，涉及程序直接控制、中断驱动、DMA 等方式。

磁盘调度的目标是**减少平均寻道时间**：

| 算法 | 特点 |
|------|------|
| FCFS | 公平，寻道距离大 |
| SSTF | 选最近请求，可能饥饿 |
| SCAN | 电梯算法，无饥饿 |
| C-SCAN | 循环扫描，响应更均匀 |

## 文件系统

文件系统负责文件的**组织、存储、检索、共享与保护**，是用户与外存之间的抽象层。需理解逻辑结构、目录结构（单级→树形）和物理结构（连续、链接、索引分配）。

## 怎么学好操作系统

1. **三轮复习**：建框架 → 攻重难点 + 刷真题 → 模拟 + 查漏补缺
2. **动手练题**：PV 操作和页面置换一定要动笔写完整过程
3. **理解优先**：多问自己「为什么引入这个概念」
4. **建立系统观**：进程需要内存，结果要存文件，过程中要 I/O —— 各模块协同服务于「让程序高效运行」
